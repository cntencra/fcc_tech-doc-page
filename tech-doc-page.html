What is Git and Github

Git is a tool for keeping track of code changes.

Git is a type of distributed version control system.

Github is a platform that stores your Git projects in an online repository.

In computer science a distributed system is a system where data or tasks are stored or carried out across multiple devices. For software version control systems this can mean a clone of the repository of code is stored on every developers local machine.

A version control system is a system that tracks any changes made to files and folders and for these changes to be easily reversible and auditable.

Using Git you can more easily create different versions of your code and work on these different versions simultaneously with other developers.

If mistakes are made and the code stops behaving as expected you can use Git to revert back to previous versions of your code.

Git allows you to work on improving parts of your code in "branches". In branches code changes can be tested without affecting the production code. Branches can allow for bug fixes that can be fully tested before being rolled out to users. Branches can also be used to create and test out new features.

Basic git workflow

Below is a basic workflow for using git on a daily basis

  1. Write codeprint("Hello world")
  2. Check status :git status

    Checking the status of your files shows which files are staged for commiting to the repository, which files are yet to be staged to the repository and which files are not being tracked by the repository.

  3. Stage changes:git add . or filename

    Staging the changes is where you select which changes are going to be added to the repository on the next commit to the repository. All changes or individual file changes or parts of file changes can be added. Keeping the commits small and of the same topic helps to make reviewing changes easier.

  4. Commit changes:git commit -m"Brief code change description" -m"more detailed description"

    Commiting the changes is where the changes are added to the local repository. This stage should include a message to detail what changes have been made to make reviewing commits easier.

  5. Push changes:git push

    Push changes is when the recent commits are pushed up to the online repository. For example Github.

Github

Github is an online repository for storing your code. the git push comand pushes changes up to an online repository. There are many different online repositories Github being one of these.

Pulling down a repository

When first starting to work on a project with other developers you will need to pull down the existing files from the repository. See below...

  1. Change Directory:git cd ../localdirectory change directory to the relevant local directory where you want the repository to be stored
  2. Clone the Repostiory:git clone html_url clone the repository
  3. Change in to the Repository Folder:cd repo_folder change into the repository folder
  4. Master Branch(master): this shows you are in the repository
  5. Checking files and folders in the Repo(master): ls -la lists all folders in the current directory including the hidden git folder
Creating a new repository

Creating a new repository maybe necessary if you are working on a new project.

Start by creating a new folder on your local machine. There can already be files in this folder that you wish to start tracking.

  1. Change Directory:git cd ../localdirectory Change directory to the relevant local directory
  2. Intialise git:git initIntialise a repository in this folder
  3. Status(master):git status Shows untracked folders and files in the repository
  4. Add(master):git add . or trackme_file Add relevant files to be commited
  5. (master):git remote add origin git@github.com:repo_html
  6. (master):git remote add origin git@github.com:repo_html
